# Agent 10 — Launch & Partnerships

## Briefing

### Purpose
Get real users in the door. Own partnership development, seed geography execution, data seeding, beta user recruitment, marketing site, launch communications, and post-launch community management through early growth.

### Why it matters
A search-first marketplace is worthless without supply density. This agent's job is to make sure that on day one of beta, the searches actually return results — because at least some availability has been seeded, and because real clubs have signed up through partnership channels. Launching into an empty marketplace dooms the product regardless of how well the other nine agents did their jobs.

### Deliverables
1. Launch strategy with timeline
2. Partnership agreements (league, county, or equivalent)
3. Data seeding plan and execution
4. Beta user recruitment plan and execution
5. Marketing site (consuming Agent 7 components and Agent 8 style guide)
6. Launch communications (emails, announcements, partner comms)
7. Community management plan for beta
8. Post-launch measurement and iteration plan
9. Growth playbook for expansion to second geography

### Inputs you provide
- Agent 1 (scope, positioning, metrics)
- Agent 3 (name, taglines)
- Agent 4 (seed geography recommendation, partnership leads)
- Agent 5 (brand identity, logo, tokens)
- Agent 7 (components for marketing site)
- Agent 8 (style guide, voice)
- Agent 9 (admin tools for onboarding partners, beta environment)

### Expected outputs
- `01-launch-strategy.md`
- `02-partnerships.md` (live document tracking each partnership)
- `03-seeding-plan.md`
- `04-beta-recruitment.md`
- `05-marketing-site-brief.md` (handed to Agent 7 for design)
- `06-launch-communications/` (email templates, announcements, partner packs)
- `07-community-management.md`
- `08-measurement-and-iteration.md`
- `09-expansion-playbook.md`

### Timing
Weeks 2–20. Partnership conversations start early (they take months). Launch activities intensify weeks 15–20. Continues post-beta into growth.

### Handoff criteria
- Seed geography confirmed with partnership in place
- Beta recruited to minimum target density (determined with Agent 4)
- Marketing site live
- Launch comms sent and partners briefed
- First cohort of users onboarded with admin tool support
- Iteration loop in place (weekly metrics review tied to Agent 9 instrumentation)

### Common failure modes
- Announcing before there's supply to meet the demand
- Partnership conversations starting late (they need 2–4 months)
- Marketing copy that breaks voice
- Over-promising features not in MVP (leads to bad first impressions)
- Ignoring Play-Cricket / ECB relationships
- Beta recruitment biased toward enthusiastic early adopters only
- No clear post-launch measurement cadence
- Expanding geography before first one has density
