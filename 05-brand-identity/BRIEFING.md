# Agent 5 — Brand Identity

## Briefing

### Purpose
Design the visual identity system: logo, colour, typography, imagery, iconography. Produce a one-page brand usage guide. Hand design tokens to the design system agent.

### Why it matters
A trust-led product lives or dies on feeling solid. A trendy or flimsy brand actively repels the fixture secretary audience. This agent's job is to produce a restrained, institutional-in-a-good-way identity without over-investing.

### Deliverables
1. Logo system (horizontal, stacked, monogram, favicon, mono variants)
2. Colour palette (primary, accent, semantic, neutral ramp) with accessibility checks
3. Typography (primary, optional secondary, licensing plan)
4. Imagery / photography guidelines
5. Iconography choice and usage rules
6. One-page brand usage guide
7. Design tokens deliverable (JSON or Figma variables) for Agent 7

### Inputs you provide
- Agent 1 positioning and voice
- Agent 3 final name
- UX review recommendations on brand
- Audience realities (age range, usage contexts)

### Expected outputs
- `01-logo-system/` (folder with all logo files and usage notes)
- `02-colour-palette.md` (with accessibility audit)
- `03-typography.md`
- `04-imagery-guidelines.md`
- `05-iconography.md`
- `06-brand-usage-guide.md` (the one-page consolidated guide)
- `07-design-tokens.json` (consumable by Agent 7)

### Timing
Weeks 3–6. Blocked until Agent 3 locks the name. Blocks Agent 7 final design system.

### Handoff criteria
- Logo system works at all required sizes including favicon
- Colour palette passes WCAG 2.2 AA for all text uses
- Design tokens delivered in machine-readable format
- Agent 7 has signed off on token handoff

### Common failure modes
- Over-elaborating the identity (brand books no one reads)
- Cricket clichés in the logo (crossed bats, stitched ball)
- Colour palettes that fail colourblind tests
- Typography that only works in headings
- Producing PDFs instead of design tokens
- "Fun" accent colours that break the tone
