# Agent 3 — Naming & Verbal Identity

## Briefing

### Purpose
Decide what the product is called and how it sounds. Maintain the canonical glossary. Block brand identity and marketing until the name is locked.

### Why it matters
Everything downstream — domain, logo, marketing — depends on the name. A wrong name is expensive to reverse. A generic name kills day-one discoverability. A trendy name ages the product before launch.

### Deliverables
1. Shortlist of 5–10 viable names with rationale and risks
2. Domain availability check (.co.uk, .com, .app, .org)
3. Companies House name clash check
4. Basic trademark sanity check (not legal advice — flag concerns)
5. Play-Cricket and ECB proximity check (too close? confusable?)
6. Final name recommendation with runner-up
7. Tagline options (3–5)
8. Product vocabulary list (feeds `shared/GLOSSARY.md`)

### Inputs you provide
- Agent 1 positioning and voice outputs
- Original scope document
- UX review recommendations on naming

### Expected outputs
- `01-naming-shortlist.md`
- `02-availability-checks.md`
- `03-name-recommendation.md`
- `04-taglines.md`
- `05-vocabulary.md` (which also updates `shared/GLOSSARY.md`)

### Timing
Weeks 2–4. Blocks Agent 5 (Brand Identity) and Agent 10 (Launch).

### Handoff criteria
- Name recommended, approved by stakeholders
- Primary domain secured
- Vocabulary merged into shared glossary
- Runner-up documented in case of late rejection

### Common failure modes
- Cricket clichés in the name (Wicket, Stumps, Googly)
- Silicon Valley vowel-soup names (Kollab, Zephyr)
- Names too close to Play-Cricket / ECB
- Failing to check domain availability until after the name is "chosen"
- Recommending a name without a runner-up
