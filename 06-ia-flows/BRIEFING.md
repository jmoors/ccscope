# Agent 6 — Information Architecture & Flows

## Briefing

### Purpose
Design how the product is organised and how users move through it. Produce sitemap, navigation, service blueprints, user journey maps, low-fidelity wireframes, and a tested clickable prototype.

### Why it matters
This is where product principles become concrete screens. If the IA is wrong, every downstream artefact is wrong. A clickable prototype tested with real users is the earliest reality check on whether the search-first model actually feels natural.

### Deliverables
1. Sitemap and navigation model
2. Full service blueprint (including edge cases from UX review)
3. User journey maps for all 8+ journeys
4. Low-fidelity wireframes for 6 priority screens
5. Clickable prototype
6. Usability test plan and results
7. IA revisions based on test findings

### Inputs you provide
- Agent 1 (scope, voice)
- Agent 2 (state machine, permissions, edge cases)
- Agent 4 (research findings, user realities)

### Expected outputs
- `01-sitemap.md`
- `02-navigation-model.md`
- `03-service-blueprint.md` (with swim lanes)
- `04-journey-maps/` (one per journey)
- `05-wireframes/` (low-fi for 6 priority screens + edge cases)
- `06-prototype-link.md` (plus Figma/prototype files)
- `07-usability-test-plan.md`
- `08-usability-test-results.md`
- `09-ia-revisions.md` (what changed after testing)

### Priority screens for MVP
1. Dashboard
2. Find a game (criteria entry)
3. Availability results (including empty state)
4. Fixture detail with "what's missing"
5. Incoming request review
6. Trust review

### Edge case flows (must be covered)
- Weather cancellation and rearrangement
- Fixture cancelled by opponent
- Block / hide a club
- New club first experience
- Dispute handling
- Multi-team fixture manager switching context

### Timing
Weeks 3–8. Wireframes start week 3 on preliminary Agent 2 outputs; prototype and testing complete by week 7.

### Handoff criteria
- All 6 priority screens wireframed and tested
- All edge cases covered in service blueprint
- Usability findings acted on or explicitly deferred
- Agent 7 has received wireframes and flows

### Common failure modes
- Designing only the happy path
- Skipping empty states
- Producing wireframes detailed enough to pre-empt visual design
- Testing with the wrong audience (tech-savvy users instead of target demographic)
- Not testing empty states (which will be 80% of early users' experience)
- Treating the prototype as a deliverable rather than a learning tool
